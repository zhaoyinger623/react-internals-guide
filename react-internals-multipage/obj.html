A3 · 核心对象速查

看源码会反复遇到这几个对象。先混个脸熟,后面各章会展开。把它们串起来就是一次更新的数据流。

对象是什么关键字段详见
ReactElementJSX 编译产物,UI 的不可变描述(虚拟 DOM 单位)type、key、props、$$typeof第一层·虚拟DOM
Fiber工作单元 + 组件实例载体,构成 Fiber 树tag、return/child/sibling、memoizedState、flags、lanes、alternateRender·Fiber
FiberRootNode整个应用的根,调度中枢current、containerInfo、pendingLanes、finishedWork0·2
Update一次状态变更的描述lane、action/payload、next(环状链表)Trigger·UpdateQueue
UpdateQueue挂在 Fiber 上的更新队列shared.pending、baseState、baseQueueTrigger·UpdateQueue
Hook函数组件里每个 Hook 的节点,串成链表memoizedState、queue、nextRender·Hooks
EffectuseEffect/Layout 的副作用节点,环状链表tag、create、destroy、deps、nextRender·Hooks/Render·变更标记
Lane优先级车道,31 位二进制掩码位运算:& | -lanesSchedule·Lane
对象之间的关系：JSX 编译成 ReactElement → 更新生成带 Lane 的 Update，并进入 Fiber 的 UpdateQueue → Scheduler 安排任务 → reconciler 遍历 Fiber、处理 Hook 并生成带 flags 的树 → commit 提交变更。
