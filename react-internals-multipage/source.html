A4 · 源码阅读路线

目标不是背文件，而是可以从一个 API 一路跟到 DOM 提交。建议使用与教程相近的 React 18/19 主分支阅读；具体文件会演进，函数职责与调用方向比行号更可靠。打开仓库后，始终从一个可复现问题出发，例如“setCount(c => c + 1) 为什么不是立即改 DOM？”

第一轮：只追主干，暂时跳过细枝末节

首次阅读只允许自己追这条调用链。遇到不认识的 helper，先在旁边写下“它输入什么、输出什么”，不要立刻钻进去；否则会在位运算和 feature flag 里失去主线。

// 从用户代码向内核移动：建议按此顺序全局搜索函数名
dispatchSetState / classComponentUpdater.enqueueSetState
  → requestUpdateLane              // 这次更新属于哪条优先级车道？
  → enqueueConcurrentHookUpdate    // Update 如何进入队列？
  → scheduleUpdateOnFiber
  → ensureRootIsScheduled          // root 选择下一批要做的 lanes
  → performConcurrentWorkOnRoot
  → renderRootConcurrent
  → workLoopConcurrent
  → performUnitOfWork              // beginWork ↓ / completeWork ↑
  → commitRoot                     // 唯一允许改宿主环境的总入口
不要把 Scheduler 和 React 调度混为一谈。ensureRootIsScheduled 是 React 根据 lanes 选择“该渲染什么”；scheduleCallback 才是把一个回调交给独立 scheduler，决定“何时获得 CPU”。两个层次会互相调用，但职责不同。

文件地图：先知道该去哪里问问题

当你在问什么优先寻找的模块/符号读完应得到的答案
Hook 的 dispatch 从哪来？ReactFiberHooks：mountState、dispatchSetState、updateReducerHook 状态如何按调用顺序存成链表，Update 如何入队、何时被消费。
更新如何到达 root？ReactFiberWorkLoop：scheduleUpdateOnFiber、markRootUpdatedFiber 上的 lane 为什么要冒泡/汇总到 root，root 为什么是调度单位。
React 为什么能中断？ReactFiberWorkLoop + Scheduler：workLoopConcurrent、shouldYield每个 Fiber 是一个工作单元；让出控制权后 WIP 如何保留下来。
组件、DOM 节点分别怎么处理？ReactFiberBeginWork / ReactFiberCompleteWork向下递阶段算 children，向上归阶段准备 host 实例并汇总 flags。
key 为什么影响状态与 DOM 复用？ReactChildFiber：reconcileChildrenArray“同 type + 同 key”是同一位置身份的核心判定；移动/删除如何标记。
DOM 到底在哪一行改？ReactFiberCommitWork → react-dom-bindings 的 HostConfigflags 如何变成 placement/update/deletion，以及平台层如何执行 DOM 操作。

第二轮：带着断点验证，而不是只看

  1. 做一个只有 useState、列表 key、useEffect 的最小应用；先预测点击一次后哪些组件会执行、哪些 DOM 会变化。
  2. 在 dispatchSetState、scheduleUpdateOnFiber、beginWork、completeWork、commitRoot 设断点或日志。每次只验证一个问题。
  3. 记录一个对象的“身份轨迹”：同一组件的 current Fiber、其 alternate、hook queue、DOM stateNode 在 render 前后分别是谁。
  4. 再制造优先级冲突：输入框更新与 startTransition 列表更新同时发生，观察低优 WIP 被打断后仍能恢复。这时再读 Lane 和 baseQueue。

每章结束后的自测

尝试画出：输入对象 → 关键数据结构变化 → 下一跳函数 → 用户看到的结果。例如读完 UpdateQueue 后，应能解释连续三次 setCount(c => c + 1) 为什么得到 +3，以及低优先级更新被跳过后如何保留。
