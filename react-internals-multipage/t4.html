4 · React 更新的四个阶段

🧭 主线setState→Update→Lane→Scheduler→Render→Effect→Commit→DOM
🗺 本章路线图

一次更新按 Trigger → Schedule → Render → Commit 依次推进。

本章先建立更新顺序和包结构的整体地图；接下来的第 5～13 章再逐步展开。

先只关注每个阶段的输入、工作和输出。Update、Lane、Fiber 等术语会在对应章节中解释：

阶段输入这一阶段做什么输出
① Trigger 触发用户调用 setState把“状态要怎样变化”记录成一条更新，并放入队列。待处理的更新
② Schedule 调度待处理的更新判断哪些更新更紧急，并安排 React 何时开始或继续工作。获得执行机会的任务
③ Render 计算任务与当前界面树执行组件、计算新状态、比较子节点，得到下一版界面和变更清单。计算完成的新树
④ Commit 提交计算完成的新树把变更应用到 DOM，并执行布局相关生命周期和 Effect。用户可见的新界面

四阶段背后的 React 包结构

四个阶段说明一次更新怎样向前推进，React 包结构说明这些工作由哪部分代码负责。先记住四个核心包：

更新阶段主要涉及的包这个包在这里负责什么
① Triggerreact + react-reconcilerreact 提供 setState、Hooks 等使用入口；Reconciler 创建 Update 并放入更新队列。
② Schedulereact-reconciler + schedulerReconciler 用 Lane 判断先处理哪些更新；scheduler 安排任务何时获得执行机会。
③ Renderreact-reconciler执行组件，计算状态，构建 Fiber 树并完成 Diff。这里主要在内存中工作。
④ Commitreact-reconciler + react-domReconciler 根据变更标记发出操作；react-dom 把结果真正应用到浏览器 DOM。
先形成这个架构印象：react 是开发者使用的入口，scheduler 负责安排时间，react-reconciler 是更新流程的核心，react-dom 负责把结果落到浏览器页面。
