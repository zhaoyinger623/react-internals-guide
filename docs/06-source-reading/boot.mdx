从 `createRoot(...).render(&lt;App/>)` 到第一次把界面画出来,React 先搭了个「地基」——创建两个根对象。很多人卡在**「为什么要两个?它们分别是啥?那些箭头又是什么?」**,先用一个类比讲清楚,再看代码和图。

### 🏗️ 先用「盖楼」类比理解三个角色

源码里的东西

类比

大白话

`#root`(DOM 容器)

一块空地

你在 HTML 里留的 `&lt;div id="root">`,界面就盖在这块地上

`fiberRootNode`

应用根对象

保存 DOM 容器、待处理的更新优先级，以及当前正在使用的 Fiber 树

`hostRootFiber`

组件树的「树根」

你写的所有组件(`&lt;App/>` 及其子孙)都长在它下面,是 Fiber 树最顶端的节点

`fiberRoot.current` 指向当前 Fiber 树的根节点；`hostRootFiber.stateNode` 反向指向 `fiberRoot`。这两个引用让调度系统可以从应用根找到当前树，也可以从树根返回应用根。

```js
// React 18 并发入口
import { createRoot } from 'react-dom/client';
const root = createRoot(document.getElementById('root'));  // ① 建两个根对象
root.render(<App/>);                                    // ② 发起首次更新
```

### 启动过程分为三步

1.  **创建两个根对象** —— `createRoot(#root)` 创建应用根 `fiberRootNode` 和 Fiber 树根 `hostRootFiber`，并建立双向引用。
2.  **把 &lt;App/> 变成一次「更新」** —— `root.render(&lt;App/>)` 生成一个 `Update`,挂到 `hostRootFiber` 的更新队列上,再喊一声 `scheduleUpdateOnFiber`(“有活了,安排渲染”)。
3.  **进入四个阶段** —— 从此走[更新主线·四个阶段](/01-foundations/t4) 的 Trigger→Schedule→Render→Commit。因为屏幕上还什么都没有,这第一次特别叫 **mount(挂载)**;以后再更新叫 update。

**与双缓存的关系：**首次渲染时，`hostRootFiber` 是 current 树的起点，React 在内存中构建 workInProgress 树。提交完成后，`fiberRoot.current` 指向新树。详见[Render·双缓存](/02-update-pipeline/t5)。

**本节过程：**创建应用根和 Fiber 树根 → 将 `&lt;App/>` 放入根节点的更新队列 → 触发首次渲染（mount）。
